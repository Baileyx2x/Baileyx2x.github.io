实验报告：基于 Serverless 架构与 Hexo 的静态博客云端部署

课程名称：云计算技术
实验名称：实验二——网站部署技能
学生姓名：薛熠阳 (Github：Baileyx2x)
日期：2026-03-25

一、 实验目的与技术背景

  1. 理解 Serverless 与静态网页托管：对比传统基于 IaaS（如自行配置云服务器的 LAMP 架构）的网站搭建方式，深入理解 GitHub Pages 提供的 Serverless 静态页面托管服务的优势与局限性。
  2. 掌握 Node.js 工具链：熟悉现代前端工程化基础，包括 NPM 包管理机制、依赖隔离（package.json）以及本地微型 Web 服务器的启动。
  3. 实践 CI/CD 雏形：通过自动化脚本实现从本地 Markdown 源码到云端 HTML 静态网页的编译、版本控制与一键部署过程。
  4. 构建技术文档基础设施：为后续的软件工程项目搭建具备高可用性、零运维成本的公网展示与文档分发平台。

二、 实验环境与工具栈

  • 操作系统：Windows 11
  • 运行环境：Node.js (LTS 版本)
  • 环境管理：nvm-windows (用于多版本 Node.js 环境的隔离与切换)
  • 版本控制与部署通道：Git (CLI)
  • 云端基础设施：GitHub (代码托管) + GitHub Pages (全球 CDN 加速与域名解析)
  • 核心框架：Hexo (基于 JavaScript 的静态站点生成器)

三、 实验核心步骤与原理分析

1. 云端资源池配置 (GitHub Pages)

在 GitHub 平台创建命名格式严格限定为 Baileyx2x.github.io 的 Public 仓库。

  • 原理分析：GitHub Pages 会自动为其底层分配独立的子域名，并将其纳入全球内容分发网络（CDN）中。该平台仅支持纯静态文件（HTML/CSS/JS）的解析，不支持 PHP、Java 等后端动态语言的执行。

2. 本地构建环境与权限配置

使用 nvm 安装 Node.js 并配置国内镜像源（npmmirror）以解决跨境网络延迟问题。

  • 关键权限突破：在初始化 Hexo 框架时，Windows PowerShell 默认的安全策略会拦截未签名的第三方脚本（抛出 SecurityError: PSSecurityException）。通过以管理员身份执行 set-ExecutionPolicy RemoteSigned，赋予本地终端执行 Node.js 自动化脚本的权限。

3. 框架初始化与依赖装载

执行 hexo init myblog 构建项目骨架，并通过 npm install 挂载依赖树。

  • 依赖管理机制：在引入第三方主题（如 Fluid 或 Redefine）及 Git 部署插件（hexo-deployer-git）时，使用 --save 参数将依赖版本号精确写入 package.json。这一机制确保了即使未来更换开发设备，只需同步配置文件即可通过 npm i 瞬间还原完整的编译环境，符合云原生开发中“应用与环境解耦”的原则。

4. 静态编译与云端推送 (Deploy)

  • 本地编译 (hexo generate)：Hexo 引擎读取 source 目录下的 Markdown 源码，结合设定的 YAML 配置文件与主题模板，经过 AST（抽象语法树）解析后，渲染输出标准的 Web 静态文件至 public 文件夹。
  • 一键部署 (hexo deploy)：触发 hexo-deployer-git 插件，在底层自动执行 Git 协议的 add、commit 与 push 操作，将 public 目录作为分支覆盖推送至远程 GitHub 仓库，完成云端迭代。

四、 实验常见问题与故障排除 (Troubleshooting)

  1. 终端脚本执行受限 (UnauthorizedAccess)
    • 现象：系统提示“无法加载文件 hexo.ps1，因为在此系统上禁止运行脚本”。
    • 解决：这是 Windows 的默认执行策略限制。通过更改 PowerShell 的 ExecutionPolicy 为 RemoteSigned 成功放行。
  2. 云端服务连通性警告 (CDN Unavailable)
    • 现象：编译时终端输出 WARN [redefine] ZStatic/CDNJS CDN is unavailable yet.
    • 分析与解决：由于本地网络到国际第三方开源 CDN 节点的路由节点存在波动或被阻断。Hexo 具备良好的容错机制，会自动降级采用备用 CDN 节点或回源拉取本地静态资源，不影响最终构建产物的完整性。

五、 实验总结与项目扩展

通过本次实验，我完整体验了静态网站“本地渲染-云端托管”的全生命周期，深刻体会到了 Serverless 架构在降低运维成本、抵御高并发访问方面的架构优势。

该云端博客不仅完成了本次实验的要求，还将直接服务于我正在进行的其他核心工程项目。例如，后续可利用该平台低成本托管 Android 应用“育芽守护”的隐私政策协议与 API 接口文档；同时，其原生支持的 Markdown 语法与代码高亮特性，也极大地便利了用于记录 CUDA 并行算法优化日志，以及沉淀 RAG (检索增强生成) 相关架构的参赛技术复盘。这种将云基础服务与实际软件工程紧密结合的实践，大幅提升了技术栈的综合利用效率。
